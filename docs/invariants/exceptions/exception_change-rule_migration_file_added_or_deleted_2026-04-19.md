# Exception: baseline file addition flagged as migration change

This exception is for a baseline snapshot file that was generated after Wave 8 migration execution. The structural change detector incorrectly flags baseline files as migration files.

## Reason

The file `schema/baselines/2026-04-19/baseline.normalized.sql` is a baseline snapshot generated by the migration verification process, not a new migration file. The structural change detector uses a broad pattern that matches baseline files in the schema directory, causing false positives for baseline additions.

- ddl | schema/baseline.sql | +: CREATE TABLE public.exchange_rate_audit_log (
- ddl | schema/baseline.sql | +: CREATE TABLE public.project_boundaries (
- ddl | schema/baseline.sql | +: CREATE TABLE public.protected_areas (
- ddl | schema/baseline.sql | +: CREATE TABLE public.statutory_levy_registry (

## Mitigation

Baseline files are generated snapshots used for verification and comparison purposes. They do not change the runtime schema and are not migrations. The structural change detector should be refined to exclude `schema/baselines/` directory from migration file detection.
