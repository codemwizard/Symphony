## Wave Verification Rule

- Wave verification rule: run `bash scripts/dev/pre_ci.sh` once after the full wave is implemented; do not run `pre_ci.sh` as a per-task closeout command inside this batch.
- Use `scripts/agent/run_task.sh <TASK_ID> --skip-bootstrap` for wave tasks to skip bootstrap.sh (which includes pre_ci.sh) during per-task execution.
- Run `bash scripts/dev/pre_ci.sh` manually once after all wave tasks are complete.

## Task Groups

### Statutory Levy Registry (TSK-P2-REG-001-*)
Tasks for creating and configuring the statutory_levy_registry table with temporal uniqueness constraints and append-only triggers.

### Exchange Rate Audit Log (TSK-P2-REG-002-*)
Tasks for creating and configuring the exchange_rate_audit_log table with high-precision rate tracking and append-only triggers.

### PostGIS Spatial Tables (TSK-P2-REG-003-*)
Tasks for installing PostGIS extension and creating spatial tables (protected_areas, project_boundaries) with DNSH and K13 taxonomy alignment triggers.

### Invariant Promotion (TSK-P2-REG-004-*)
Tasks for verifying check_reg26_separation() function and promoting INV-169 to implemented status.

### State Machine Enforcement (TSK-P2-PREAUTH-005-*)
Tasks for creating state machine enforcement layer with state_transitions and state_current tables, plus trigger functions for transition authority, signature, execution binding, and state update rules.
